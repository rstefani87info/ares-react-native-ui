# Checklist Migliorie aReS React Native UI

## Alta Priorita

- [X] Correggere l'entrypoint del package in `package.json`: oggi `"main": "expo-router/entry"` ma `expo-router` non e' dichiarato tra le dipendenze; l'import del package lato consumer risulta fragile/non risolvibile.
- [X] Allineare le dipendenze reali del runtime: i moduli importati dal codice del package vanno dichiarati in `dependencies` (es. `react-native-gesture-handler`, `react-hook-form`, `yup`, `axios`, `react-native-device-info`).
- [X] Sistemare `effects/query.js`: implementazione hook `query()` non consistente con `axios` (usa `response.ok`/`response.json()` e ritorna `data` non definito), rischio di crash immediato in app consumer.
- [X] Stabilizzare il supporto a `@ares/react-native-ui-dev`: `ApplicationRoot` puo' leggere `./.tmp/ares-rn-args.json` (chiave `path|component|render`) per renderizzare un componente passato da riga di comando, senza log/side effects quando non configurato.

## Contratto Modulo (API Pubbliche)

- [X] Definire la superficie pubblica stabile del package: oggi la documentazione suggerisce import profondi (`@ares/react-native-ui/components/...`) e il codice interno fa self-import sul package name; formalizzare gli entrypoint supportati.
- [X] Introdurre un entrypoint "root" per l'uso normale (es. export di `ApplicationRoot`, `ARESProvider`, `AuthProvider`, `setConfig`, componenti principali) evitando deep import come default di onboarding.
- [X] Stabilire convenzioni di configurazione: chiarire come viene iniettata l'istanza `aReS` (`config.ares`), come vengono inizializzati i datasource e quali hook opzionali sono attesi (es. `aReS.contextSettings.auth.*`).

## Packaging

- [X] Aggiungere una mappa `exports` in `package.json` (coerente con lo stile dei moduli `core`/`web`) per esplicitare root e sottopath pubblici e ridurre import fragili.
- [X] Valutare e dichiarare i campi tipici RN: `react-native`, `module`, `types` (o dichiarazioni `.d.ts` minime), e una strategia chiara su JSX/ESM (Metro vs Node resolution).
- [X] Aggiungere `files`/`.npmignore` per non pubblicare artefatti non necessari e per evitare che path locali o file temporanei entrino nel pacchetto.

## Dipendenze e Peer Dependencies

- [X] Spostare `react` e `react-native` a `peerDependencies` (con range compatibili) per evitare istanze duplicate in app consumer.
- [X] Esplicitare peer deps tipiche di RN quando usate: `react-native-gesture-handler`, `react-native-reanimated` (richiesto indirettamente da `react-native-reanimated-carousel`), eventuali dependency native (maps/webview/vector-icons) con note di linking.
- [X] Sistemare dipendenze `@ares/web`/`@ares/web-ui`: verificare se sono davvero necessarie al runtime mobile (attualmente `@ares/web` appare usato in `effects/query.js`); in caso contrario spostarle ad opzionali o rimuoverle per ridurre peso e coupling.

## Robustezza Runtime

- [X] Eliminare o rendere opt-in i `console.log`/`console.warn` dentro componenti e context (es. `ApplicationRoot`, `AuthContext`), per evitare rumore e potenziali leak.
- [X] Evitare effetti globali non documentati (es. `global.sessionID` in `ARESContext`): definire un contratto esplicito e/o confinare lo stato nel context.
- [X] Verificare che i componenti critici non assumano che un provider sia sempre presente (fallback sensati, errori chiari quando manca `config.ares` o `ARESProvider`).

## Qualita' (Test e Lint)

- [X] Rendere eseguibili test minimi del package (smoke test): render di `ApplicationRoot`, `ARESProvider`, un campo `Text`, e un componente output (es. `Modal`) con `react-test-renderer` o `@testing-library/react-native`.
- [X] Aggiungere/standardizzare configurazione Jest (se necessaria) e assicurare che `yarn workspace @ares/react-native-ui test` sia stabile anche senza ambiente Android/iOS.
- [X] Verificare che `eslint` non richieda toolchain native per girare e che le regole siano coerenti con il resto del monorepo.

## Documentazione e Adozione

- [X] Allineare `docs/en/react-native-ui.md` e `docs/it/react-native-ui.md` al contratto reale: esempio consigliato con `setConfig({ ares })`, `ARESProvider`, `AuthProvider`, e import dagli entrypoint stabili.
- [X] Documentare prerequisiti RN non banali: setup `react-native-gesture-handler`, `reanimated` (Babel plugin/installation), `maps` (API key / linking), `google-signin` (config native), `vector-icons`.
- [X] Dichiarare chiaramente cosa e' "libreria UI" vs "app template/dev harness" (in particolare se esiste un workspace dedicato tipo `react-native-ui-dev`).
